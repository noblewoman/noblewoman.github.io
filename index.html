sourcetree 推送问题

  • 背景：私有库里接入了腾讯云短视频SDK，搞完配置要推送到远程时，推送失败，提示：POST Git-receive-pack (chunked)。

  • 原因：网上查找到一个比较靠谱的说法：This is a bug in git, when using https it will use chunked encoding for uploads above a certain size. those do not work.
    A trivial fix is to tell git to not chunk until some ridiculously large size value, such as: git config http.postBuffer 524288000

  • 解决方法：1. 命令行配置：git config http.postBuffer 524288000
    2.打开sourcetree -> 设置（右上角） -> 高级 -> 编辑配置文件
    在文件最后添加
    [http] 

    postBuffer = 524288000

    点击确定，重新推送。

  • 后续问题，sourcetree 推送之后：提示：POST git-receive-pack (239851501 bytes) …

  • 解决：转成命令推送 git push origin master
    命令卡在：total 41(delta 8), reused 0(delta 0)
  • 把原来修改的配置文件删掉。推送成功。
